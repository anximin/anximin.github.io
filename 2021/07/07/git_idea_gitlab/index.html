IDEA集成GitLab版本控制

环境准备

  • 本地开发环境安装好了git
  • IDEA 可正常使用
  • 已经创建好了gitlab项目

检验本地git程序状态

依次选择 File => Settings => Version Control => Git


path to Git executable: 填写本地的Git程序路径，单机test进行测试。


出现Git 版本的提示证明配置成功了。

项目的管理 【远程克隆到本地】&【本地导入远程库】

克隆远程库（远程项目导入）到本地
VCS => Checkout From Version Control => Git


之后根据提示，输入gitlab上的用户名与密码，即可完成项目的导入

本地项目发布到远程库

  1. VCS => Import into Version Control => Create Git Repository...
    此时项目的文件会变成红色

  2. 在项目单击右键 => Git => Add
    然后项目的文件会变成绿色

  3. 在项目单击右键 => Git => Commit Directory...

    1. 在弹出的Commit Changes 窗口中 选中项目文件夹以选中所有文件
    2. 在 Commit Message 窗口中 输入本次提交的版本信息
    3. 单击右下角的 Commit 按钮。

  4. 在项目单击右键 => Git => Repository => Push...

    1. 在弹出的 Push Commits 窗口中 单击对应项目名称后面的 Define remote.
    2. 输入GitLab上项目名称和远程仓库的地址。
    3. 点击 Push 完成项目的发布。
